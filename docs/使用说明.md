# Poi 图像识别自动化框架使用说明

这是一个运行在 Windows 上的 Poi / 舰娘 Collection 自动化脚本框架。当前版本支持：

- 截取 Poi 浏览器内纯净的 `1200x720` 游戏画面。
- 实时预览游戏画面并显示鼠标逻辑坐标。
- 使用手动提供的模板图识别当前页面。
- 根据当前页面显示可点击的功能入口按钮。
- 通过 YAML 规则驱动任务状态机和点击坐标。

本项目不使用 AI，不接入 OCR 或自动学习。所有页面、入口、按钮、阵型、血量判断都通过 GUI 配置、YAML 规则和模板图手动维护。

## 安装

建议使用 Python 3.11 或更新版本。

```powershell
cd E:\Codex_project\Poi图像识别自动化
python -m venv .venv
.\.venv\Scripts\Activate.ps1
python -m pip install -e .
```

启动 GUI：

```powershell
poi-auto
```

也可以直接运行：

```powershell
python -m poi_auto.app.main
```

## 第一次使用

1. 启动 Poi 浏览器和本软件。
2. 在左侧“目标窗口”里填写标题关键字，例如 `poi`。
3. 点击“刷新窗口”，从候选窗口里选择真正的 Poi 浏览器窗口。
4. 保持“排除本软件窗口”开启。
5. 点击“保存配置”。
6. 右侧实时预览应显示纯净游戏画面。

如果截图不对，先调整标题关键字和目标窗口；如果画面有少量偏移，在 GUI 里调整 `offset_x` 或 `offset_y`。

## 截图区域

默认配置在 `config/default.yaml`：

```yaml
game:
  logical_width: 1200
  logical_height: 720
  capture_width: 1200
  capture_height: 720
  offset_x: 0
  offset_y: 0
  crop_mode: left_center_fixed
```

`left_center_fixed` 会在 Poi 窗口客户区内，从左侧开始、垂直居中裁出固定 `1200x720` 游戏画面。

裁剪公式：

```text
x = client.left + offset_x
y = client.top + (client.height - capture_height) / 2 + offset_y
```

所有点击坐标都按 `1200x720` 逻辑坐标填写。

## 实时预览和坐标采集

实时预览默认每 `200ms` 刷新一次。鼠标在游戏区域内时，预览图会显示红色十字、圆环和当前逻辑坐标，例如 `520, 310`。

这个坐标可以直接用于：

- `config/pages.yaml` 的功能入口坐标。
- `config/tasks/sortie.yaml` 的任务点击点坐标。

## 页面识别

页面识别规则在 `config/pages.yaml`。

页面模板统一放在：

```text
assets/templates/pages/
```

示例：

```yaml
monitor:
  interval_ms: 500
  unknown_threshold: 0.75

pages:
  home:
    name: "母港"
    min_matches: 1
    templates:
      - path: "home/sortie_button.png"
        threshold: 0.86
      - path: "home/resource_bar.png"
        threshold: 0.82
    actions:
      sortie:
        name: "出击"
        x: 310
        y: 560
```

识别逻辑：

- 每个页面可以配置多个模板。
- 每个模板可以单独设置 `threshold`。
- 页面达到 `min_matches` 后才算命中。
- 多个页面同时命中时，选择平均分最高的页面。
- 没有页面命中时显示 `unknown`。
- 模板缺失不会崩溃，会在 GUI 的模板详情里显示 `missing_template`。

可选的局部搜索区域：

```yaml
templates:
  - path: "home/sortie_button.png"
    threshold: 0.86
    region:
      x: 150
      y: 420
      width: 300
      height: 220
```

## 功能入口按钮

当 GUI 识别到已知页面后，会在“当前页面功能入口”区域显示该页面的 `actions`。

例如：

```yaml
actions:
  sortie:
    name: "出击"
    x: 310
    y: 560
  supply:
    name: "补给"
    x: 120
    y: 500
```

点击 GUI 中的“出击”按钮后，程序会按当前游戏裁剪区域把逻辑坐标换算成屏幕坐标并点击。

## 出击规则

出击规则在 `config/tasks/sortie.yaml`。

状态可以继续使用模板，也可以使用通用页面识别：

```yaml
home:
  detect:
    type: "page"
    page: "home"
  actions:
    - type: "click"
      point: "sortie_entry"
    - type: "wait"
      ms: 800
  next: "map_select"
```

支持的动作：

- `log`：写日志。
- `click`：点击 `points` 中配置的逻辑坐标。
- `wait`：等待指定毫秒。
- `formation`：点击当前配置阵型。
- `damage_check`：执行血量规则预留逻辑。
- `stop`：停止当前任务。

## 模板目录建议

```text
assets/templates/pages/home/
assets/templates/pages/sortie/
assets/templates/pages/formation/
assets/templates/sortie/damage/
```

页面识别模板放在 `assets/templates/pages/` 下；任务专用模板可以继续放在任务目录下。

## 后续扩展

新增任务建议按下面方式组织：

```text
src/poi_auto/tasks/repair/
src/poi_auto/tasks/resupply/
src/poi_auto/tasks/expedition/
config/tasks/repair.yaml
config/tasks/resupply.yaml
config/tasks/expedition.yaml
```

约定：

- `device/` 只处理窗口、截图和点击。
- `vision/` 只处理模板匹配和页面识别。
- `tasks/` 只处理游戏流程。
- `gui/` 只处理界面展示和按钮交互。
- `core/` 只处理配置、运行上下文和任务线程。
