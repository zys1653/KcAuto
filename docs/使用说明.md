# Poi 图像识别自动化框架使用说明

这是一个运行在 Windows 上的 Poi / 舰娘 Collection 自动化脚本框架。当前版本支持纯净游戏画面截图、页面模板识别、Alas 风格三栏 GUI、页面动作测试，以及出击任务的多战斗阵型选择流程。

本项目不使用 AI，不接入 OCR 或自动学习。所有页面、入口、按钮、地图、阵型、血量判断都通过 GUI 配置、YAML 规则和模板图手动维护。

## 安装与启动

建议使用 Python 3.11 或更新版本。

```powershell
cd E:\Codex_project\Poi图像识别自动化
python -m venv .venv
.\.venv\Scripts\Activate.ps1
python -m pip install -e .
poi-auto
```

也可以直接运行：

```powershell
python -m poi_auto.app.main
```

## GUI 布局

界面分为三栏：

- 左侧：一级功能导航，包含出击、演习、远征、补给、入渠、软件调试。
- 中间：当前功能的设置页。
- 右侧：始终显示实时截图和日志。

顶部只保留任务下拉框、启动任务、停止任务。当前只有“出击”任务实现了自动化逻辑，其他任务先显示预留页面。

## 出击设置

左侧选择“出击”后，可以配置：

- 出击海域：`1-1` 到 `1-5`、`2-1` 到 `2-5`、`3-1` 到 `3-5`。
- 阵型选择：单纵阵、复纵阵、轮形阵、梯形阵、单横阵。
- 最大战斗次数：默认 `6`，用于防止状态识别异常后无限循环。
- 大破撤退：开启后如果大破模板命中，会撤退并尝试回母港。

这些设置会保存到 `config/default.yaml`：

```yaml
sortie:
  map: 1-1
  formation: line_ahead
  max_battles: 6
  stop_on_heavy_damage: true
```

## 软件调试

左侧选择“软件调试”后，可以配置：

- 目标窗口标题关键字和候选窗口。
- 截图模式、逻辑分辨率、固定截图区域和偏移量。
- 点击延迟、移动耗时、模板匹配阈值。
- 实时预览、鼠标位置高亮、刷新间隔。
- 停止快捷键。
- 页面识别状态、模板详情和当前页面动作测试按钮。

## 截图区域

默认截图配置：

```yaml
game:
  logical_width: 1200
  logical_height: 720
  capture_width: 1200
  capture_height: 720
  offset_x: 0
  offset_y: 0
  crop_mode: left_center_fixed
```

`left_center_fixed` 会在 Poi 窗口客户区内，从左侧开始、垂直居中裁出固定 `1200x720` 游戏画面。所有点击坐标都按这个逻辑坐标填写。

## 页面识别

页面识别规则在 `config/pages.yaml`。页面模板统一放在：

```text
assets/templates/pages/
```

示例：

```yaml
pages:
  home:
    name: "母港"
    min_matches: 2
    templates:
      - path: "home/sortie_button.png"
        threshold: 0.86
      - path: "home/resource_bar.png"
        threshold: 0.82
    actions:
      sortie:
        name: "出击"
        x: 327
        y: 401
```

页面达到 `min_matches` 后才算命中。多个页面同时命中时，选择平均分最高的页面。

## 出击规则

出击规则在 `config/tasks/sortie.yaml`。地图规则放在 `maps` 区：

```yaml
maps:
  "1-1":
    name: "1-1"
    detect:
      type: "template"
      template: "sortie/maps/1-1.png"
      threshold: 0.86
    select: { x: 180, y: 220 }
    start: { x: 1020, y: 650 }
```

地图模板放在：

```text
assets/templates/sortie/maps/
```

阵型和通用按钮坐标也在 `sortie.yaml`：

```yaml
formation_points:
  line_ahead: { x: 648, y: 279, name: "单纵阵" }

points:
  return_home: { x: 55, y: 55, name: "母港" }
  proceed: { x: 790, y: 600, name: "进击" }
  retreat: { x: 410, y: 600, name: "撤退" }
```

## 出击流程

当前出击任务流程：

1. 识别母港 `home`。
2. 点击母港出击入口。
3. 识别出击菜单 `sortie_menu`。
4. 按 `sortie.map` 读取 `sortie.yaml maps[map]`。
5. 检测并点击对应海域。
6. 点击开始出击。
7. 每次识别到 `formation` 页面，都点击当前配置阵型，并递增战斗次数。
8. 识别到 `advance_or_retreat` 页面时，默认进击。
9. 达到最大战斗次数或大破模板命中时，点击撤退和返回母港。
10. 回到母港后停止任务。

如果当前页面未知、模板缺失、地图未配置或关键坐标缺失，任务会停止并写日志，不会继续盲点。

## 模板建议

建议逐步补齐这些模板：

```text
assets/templates/pages/home/
assets/templates/pages/sortie/
assets/templates/pages/formation/
assets/templates/pages/battle/
assets/templates/sortie/maps/
assets/templates/sortie/damage/
```

先确保母港、出击菜单、阵型选择、进击/撤退页面能稳定识别，再扩展战斗结果和血量检测。
