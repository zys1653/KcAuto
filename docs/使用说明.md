# Poi 图像识别自动化框架使用说明

这是一个运行在 Windows 上的 Poi / 舰娘 Collection 自动化脚本框架。当前版本支持纯净游戏画面截图、页面模板识别、Alas 风格三栏 GUI、页面动作测试，以及出击任务的页面驱动流程。

本项目不使用 AI 或自动学习。所有页面、入口、按钮、地图、阵型、损伤判断都通过 GUI 配置、YAML 规则和模板图手动维护。HP 数字 OCR 只是可选日志记录，不参与大破撤退判断。

## 安装与启动

建议使用 Python 3.11 或更新版本。

```powershell
cd E:\Codex_project\Poi图像识别自动化
python -m venv .venv
.\.venv\Scripts\Activate.ps1
python -m pip install -e .
poi-auto
```

也可以直接运行：

```powershell
python -m poi_auto.app.main
```

## GUI 布局

界面分为三栏：

- 左侧：一级功能导航，包含出击、演习、远征、补给、入渠、软件调试。
- 中间：当前功能的设置页。
- 右侧：始终显示实时截图和日志。

顶部只保留任务下拉框、启动任务、停止任务。当前只有“出击”任务实现了自动化逻辑，其他任务先显示预留页面。

## 出击设置

左侧选择“出击”后，可以配置：

- 出击海域：`1-1` 到 `1-5`、`2-1` 到 `2-5`、`3-1` 到 `3-5`。
- 阵型选择：单纵阵、复纵阵、轮形阵、梯形阵、单横阵。
- 最大战斗次数：默认 `6`，用于防止状态识别异常后无限循环。
- 大破撤退：开启后如果大破模板命中，会撤退并尝试回母港。
- 记录 HP OCR：开启后尝试把 HP 数字写入日志；OCR 不参与撤退判断，缺少 OCR 依赖时会跳过。

这些设置会保存到 `config/default.yaml`：

```yaml
sortie:
  map: 1-1
  formation: line_ahead
  max_battles: 6
  stop_on_heavy_damage: true
  hp_ocr_enabled: true
```

## 软件调试

左侧选择“软件调试”后，可以配置：

- 目标窗口标题关键字和候选窗口。
- 截图模式、逻辑分辨率、固定截图区域和偏移量。
- 点击延迟、移动耗时、模板匹配阈值。
- 实时预览、鼠标位置高亮、刷新间隔。
- 停止快捷键。
- 页面识别状态、模板详情和当前页面动作测试按钮。

## 截图区域

默认截图配置：

```yaml
game:
  logical_width: 1200
  logical_height: 720
  capture_width: 1200
  capture_height: 720
  offset_x: 0
  offset_y: 0
  crop_mode: left_center_fixed
```

`left_center_fixed` 会在 Poi 窗口客户区内，从左侧开始、垂直居中裁出固定 `1200x720` 游戏画面。所有点击坐标都按这个逻辑坐标填写。

## 页面识别

页面识别规则在 `config/pages.yaml`。页面模板统一放在：

```text
assets/templates/pages/
```

示例：

```yaml
pages:
  home:
    name: "母港"
    min_matches: 2
    templates:
      - path: "home/sortie_button.png"
        threshold: 0.86
      - path: "home/resource_bar.png"
        threshold: 0.82
    actions:
      sortie:
        name: "出击"
        x: 327
        y: 401
```

页面达到 `min_matches` 后才算命中。多个页面同时命中时，选择平均分最高的页面。

## 出击规则

出击规则在 `config/tasks/sortie.yaml`。这个文件现在只描述出击流程，不直接保存页面按钮坐标。页面按钮坐标统一写在 `config/pages.yaml` 的 `actions` 里，出击规则通过页面 key 和 action key 引用它们。

入口流程示例：

```yaml
entry_flow:
  - page: home
    action: sortie
    wait_ms: 800
  - page: sortie_menu
    action: sortie_start
    wait_ms: 800
  - page: map_select_1
    action_from_map: true
    wait_ms: 800
  - page: sortie_confirm
    action: decide
    wait_ms: 1200

map_actions:
  "1-1": { page: map_select_1, action: map_1_1 }
```

页面动作示例在 `pages.yaml`：

```yaml
pages:
  map_select_1:
    actions:
      map_1_1:
        name: "1-1"
        x: 405
        y: 298
```

循环页面规则示例：

```yaml
loop_pages:
  compass:
    action: spin
    wait_ms: 3000
  formation:
    action_type: choose_formation
  battle:
    action_type: collect_damage
    click: false
  advance_or_retreat:
    action_type: advance_or_retreat
```

## 出击流程

当前出击任务流程：

1. 识别母港 `home`。
2. 按 `entry_flow` 点击母港出击入口。
3. 识别出击菜单 `sortie_menu`。
4. 按 `map_actions[sortie.map]` 点击 `pages.yaml` 里对应的地图 action。
5. 点击决定并进入出击循环。
6. 识别到 `compass` 时点击罗盘并等待。
7. 识别到 `formation` 时点击当前配置阵型，并递增战斗次数。
8. 识别到 `battle` 时只采集损伤状态，不点击。
9. 识别到夜战、战斗评价、经验结算、获得新船或补给点时，按 `loop_pages` 配置点击推进。
10. 识别到 `advance_or_retreat` 时，根据大破检测和最大战斗次数决定进击或撤退。
11. 识别到 `return_home` 或未知页面时结束任务。

如果当前页面未知、模板缺失、地图动作未配置或关键 action 缺失，任务会停止并写日志，不会继续盲点。当前示例配置只提供第 1 世界选图页的动作；其他海域需要先在 `pages.yaml` 增加页面和 action，再在 `sortie.yaml map_actions` 里引用。

## 模板建议

建议逐步补齐这些模板：

```text
assets/templates/pages/home/
assets/templates/pages/sortie/
assets/templates/pages/formation/
assets/templates/pages/battle/
assets/templates/sortie/damage/
```

先确保母港、出击菜单、地图选择、出击决定、阵型选择、进击/撤退页面能稳定识别，再扩展罗盘、夜战、战斗结果、经验结算、新船、补给点和损伤检测。
