# Poi 图像识别自动化框架使用说明

这是一个运行在 Windows 上的 Poi / 舰娘 Collection 自动化脚本框架。当前版本支持固定游戏区域截图、页面模板识别、三栏 GUI、页面动作测试，以及出击任务的页面驱动流程。

本项目不使用 AI 或自动学习。所有页面、入口、按钮、地图、阵型、损伤判断都通过 GUI 配置、YAML 规则和模板图手动维护。HP 数字 OCR 只是可选日志记录，不参与大破撤退判断。

## 安装与启动

建议使用 Python 3.11 或更新版本。

```powershell
cd E:\Codex_project\Poi图像识别自动化
python -m venv .venv
.\.venv\Scripts\Activate.ps1
python -m pip install -e .
poi-auto
```

也可以直接运行：

```powershell
$env:PYTHONPATH = "src"
python -m poi_auto.app.main
```

如果需要 HP OCR，请额外安装 Python 包和本机 Tesseract：

```powershell
python -m pip install pytesseract
```

Tesseract 本体需要单独安装，并确保 `tesseract.exe` 在系统 `PATH` 中。没有 OCR 依赖时，程序仍可正常进行模板识别和出击流程，只是 HP 日志会提示不可用。

## GUI 布局

界面分为三栏：

- 左侧：一级功能导航，包含出击、演习、远征、补给、入渠、软件调试。
- 中间：当前功能的设置页。
- 右侧：实时截图和带颜色的日志。

顶部保留任务下拉框、启动任务、停止任务。当前只有“出击”任务实现了自动化逻辑，其他任务先显示预留页面。

日志颜色大致含义：

- 浅灰：普通信息。
- 橙黄：warning、未知页面、OCR 不匹配或不可用。
- 红色：失败、异常、停止、缺少配置。
- 绿色：保存、启动、完成等成功状态。

## 出击设置

左侧选择“出击”后，可以配置：

- 出击海域：当前配置支持第 1 世界普通图 `1-1` 到 `1-5`。
- 阵型选择：单纵阵、复纵阵、轮形阵、梯形阵、单横阵。
- 最大战斗次数：用于防止状态识别异常后无限循环。
- 大破撤退：开启后如果大破模板命中，会撤退。
- 记录 HP OCR：开启后在战斗中尝试把我方 HP 数字写入日志；OCR 不参与撤退判断。
- 舰娘数量：本次出击舰娘数量，范围 `1-6`。如果 OCR 识别到的 HP 行数不一致，日志会输出 warning。

这些设置会保存到 `config/default.yaml`：

```yaml
sortie:
  map: 1-1
  formation: line_ahead
  max_battles: 6
  stop_on_heavy_damage: true
  hp_ocr_enabled: true
  ship_count: 6
```

## 软件调试

左侧选择“软件调试”后，可以配置和查看：

- 目标窗口标题关键字和候选窗口。
- 截图模式、逻辑分辨率、固定截图区域和偏移量。
- 点击延迟、移动耗时、模板匹配阈值。
- 实时预览、鼠标位置高亮、刷新间隔。
- 页面识别间隔。
- 停止快捷键。
- HP OCR 区域、边距、放大倍数和每秒识别次数。
- 页面识别状态、模板详情和当前页面动作测试按钮。

调试页内容较多，已经放进滚动区域。如果中间设置显示不全，可以用鼠标滚轮滚动。

## 截图区域

默认截图模型：

```yaml
game:
  logical_width: 1200
  logical_height: 720
  capture_width: 1200
  capture_height: 720
  offset_x: 0
  offset_y: 0
  crop_mode: left_center_fixed
```

`left_center_fixed` 会在 Poi 窗口客户区内，从左侧开始、垂直居中裁出固定游戏画面。所有点击坐标、模板区域和 OCR 区域都按逻辑坐标填写。

如果预览没有完整覆盖游戏画面：

1. 先确认 Poi 窗口足够大。
2. 用软件调试页调整 `capture_width` / `capture_height`。
3. 用 `offset_x` / `offset_y` 微调裁剪位置。
4. 观察右侧预览和鼠标逻辑坐标是否符合实际游戏画面。

## 页面识别

页面识别规则在 `config/pages.yaml`。页面模板统一放在：

```text
assets/templates/pages/
```

示例：

```yaml
pages:
  map_select_1:
    name: "地图1普通选关"
    min_matches: 3
    templates:
      - path: "sortie/fight_menu_1_1.png"
        threshold: 0.86
        required: true
      - path: "sortie/fight_menu_1_2.png"
        threshold: 0.82
      - path: "sortie/fight_menu.png"
        threshold: 0.82
    actions:
      map_1_1:
        name: "1-1"
        x: 405
        y: 298
```

页面达到 `min_matches` 后才算命中。多个页面同时命中时，选择平均分最高的页面。

如果某个模板必须命中，可以在该模板上加 `required: true`。例如 3 个模板里要求总命中 2 个，但标题模板必须命中：

```yaml
pages:
  sample_page:
    name: "示例页面"
    min_matches: 2
    templates:
      - path: "sample/title.png"
        threshold: 0.86
        required: true
      - path: "sample/button.png"
        threshold: 0.82
      - path: "sample/icon.png"
        threshold: 0.82
```

不写 `required` 时保持默认逻辑，也就是 3 个模板任选命中 2 个即可。

`actions` 是该页面可点击的入口。出击任务和调试页动作按钮都会从这里读取坐标，所以任务配置里一般不直接写页面按钮坐标。

## 出击规则

出击规则在 `config/tasks/sortie.yaml`。这个文件描述出击流程，不直接保存页面按钮坐标。页面按钮坐标统一写在 `config/pages.yaml` 的 `actions` 里，出击规则通过页面 key 和 action key 引用它们。

入口流程示例：

```yaml
entry_flow:
  - page: home
    action: sortie
    wait_ms: 800
  - page: sortie_menu
    action: sortie_start
    wait_ms: 800
  - page: map_select_1
    action_from_map: true
    wait_ms: 800
  - page: sortie_confirm
    action: decide
    wait_ms: 1200

map_actions:
  "1-1": { page: map_select_1, action: map_1_1 }
  "1-2": { page: map_select_1, action: map_1_2 }
```

循环页面规则示例：

```yaml
loop_pages:
  compass:
    action: spin
    wait_ms: 15000
  formation:
    action_type: choose_formation
    count_battle: true
    wait_ms: 1200
  battle:
    action_type: collect_damage
    click: false
    poll_ms: 500
    transition_wait_ms: 3000
  advance_or_retreat:
    action_type: advance_or_retreat
    wait_ms: 1000
```

## 出击流程

当前出击任务流程：

1. 识别母港 `home`。
2. 按 `entry_flow` 点击母港出击入口。
3. 识别出击菜单 `sortie_menu`。
4. 按 `map_actions[sortie.map]` 点击 `pages.yaml` 里对应的地图 action。
5. 点击决定并进入出击循环。
6. 识别到 `sortie_start` 时点击开始。
7. 识别到 `compass` 时点击罗盘并等待。
8. 识别到 `formation` 时点击当前配置阵型，并递增战斗次数。
9. 识别到 `battle` 时只采集损伤状态和 HP OCR，不点击。
10. 从 `battle` 切换到其他页面后，先等待 `transition_wait_ms`，再重新截图识别。
11. 识别到夜战、战斗评价、经验结算、获得新船或补给点时，按 `loop_pages` 配置点击推进。
12. 识别到 `advance_or_retreat` 时，根据大破检测和最大战斗次数决定进击或撤退。
13. 识别到 `return_home` 后结束任务。

容错行为：

- 未知页面不会立刻停止，会按最近一次等待时间重试。
- 默认连续 3 次仍未知才停止并写日志。
- 入口流程中如果当前页面可识别但和预期不一致，程序会按当前页面在 `entry_flow` 里的配置重新点击一次，而不是一直等待用户手动切换。
- 如果地图动作、页面 action 或模板配置缺失，任务会停止并写日志，不会继续盲点。

当前示例配置主要覆盖第 1 世界选图页。其他海域需要先在 `pages.yaml` 增加页面和 action，再在 `sortie.yaml map_actions` 里引用。

## HP OCR 设置

HP OCR 用于在战斗中输出我方舰娘血量，例如：

```text
战斗中血量：1:33/33, 2:18/18, 3:17/35
```

OCR 配置保存在 `config/default.yaml`：

```yaml
ocr:
  hp_region:
    x: 250
    y: 126
    width: 90
    height: 374
  hp_padding:
    top: 8
    right: 4
    bottom: 4
    left: 4
  hp_scale: 2
  hp_rate_per_sec: 2
```

参数含义：

- `hp_region.x/y/width/height`：血量文字所在矩形区域，按游戏逻辑坐标填写。
- `hp_padding.top/right/bottom/left`：OCR 前把区域向四周额外扩展，第一行漏识别时优先加大 `top` 或把 `y` 往上调。
- `hp_scale`：OCR 前把裁剪图放大，通常试 `2` 或 `3`。
- `hp_rate_per_sec`：战斗中每秒最多 OCR 次数。它不影响页面识别速度。

如果你已知四个点是 `(250,126)`、`(340,126)`、`(250,500)`、`(340,500)`，应填写：

```yaml
hp_region:
  x: 250
  y: 126
  width: 90
  height: 374
```

因为 `width = 340 - 250 = 90`，`height = 500 - 126 = 374`。

如果第一行 HP 经常识别不到，优先尝试：

```yaml
hp_region:
  x: 240
  y: 110
  width: 110
  height: 400
hp_padding:
  top: 16
  right: 8
  bottom: 8
  left: 8
hp_scale: 3
```

调试顺序建议：

1. 先把 `y` 往上调，让第一行完整进入区域。
2. 再增加 `padding.top`，避免第一行贴边被预处理吃掉。
3. 如果数字太小，把 `hp_scale` 从 `2` 调到 `3`。
4. 如果行数少于出击页设置的舰娘数量，看 warning 里的 `actual` 和 `hp=` 判断是哪几行漏了。

## 模板建议

页面模板建议逐步补齐：

```text
assets/templates/pages/home/
assets/templates/pages/sortie/
assets/templates/pages/formation/
assets/templates/pages/battle/
assets/templates/pages/map/
```

损伤检测模板放在：

```text
assets/templates/sortie/damage/
```

先确保母港、出击菜单、地图选择、出击决定、出击开始、阵型选择、战斗中、进击/撤退页面能稳定识别，再扩展罗盘、夜战、战斗结果、经验结算、新船、补给点和回港页面。

模板调试建议：

- 阈值过高会识别不到，过低可能误判。
- 固定元素适合做页面模板，例如标题、固定按钮、固定图标。
- 容易变化的文字、资源数、背景动画不适合作为模板。
- 同一页面用多个小模板组合通常比一个大模板更稳。
- 关键模板可以加 `required: true`，降低相似页面误判。
