# Poi 图像识别自动化框架使用说明

这是一个运行在 Windows 上的 Poi / 舰娘 Collection 自动化脚本框架。当前版本的目标是搭好可扩展骨架：识别 Poi 浏览器左半侧 `1200x720` 游戏画面，通过手写 YAML 规则执行截图、状态判断和鼠标点击。

本项目不使用 AI，不接入自动学习。所有入口、按钮、阵型、血量判断都通过 GUI 配置、YAML 规则和模板图手动维护。

## 安装

建议使用 Python 3.11 或更新版本。

```powershell
cd E:\Codex_project\Poi图像识别自动化
python -m venv .venv
.\.venv\Scripts\Activate.ps1
python -m pip install -e .
```

启动 GUI：

```powershell
poi-auto
```

也可以直接运行：

```powershell
python -m poi_auto.app.main
```

## 第一次使用

1. 启动 Poi 浏览器和本软件。
2. 在左侧“目标窗口”里填写标题关键字，例如 `poi`。
3. 点击“刷新窗口”，从候选窗口里选择真正的 Poi 浏览器窗口。
4. 保持“排除本软件窗口”开启，避免截图抓到控制器自己。
5. 点击“保存配置”。
6. 右侧实时预览应显示 Poi 游戏画面；也可以点击“刷新截图”立即刷新一次。

如果截图仍然不对，先把标题关键字改短或清空，再刷新候选窗口并重新选择。

## GUI 配置

常用配置现在都可以直接在 GUI 中修改并保存到 `config/default.yaml`。

可配置项包括：

- 目标窗口标题关键字。
- 精确目标窗口标题。
- 是否排除本软件窗口。
- 截图模式：默认 `left_center_fixed`，也保留 `left_half` 和 `full`。
- 游戏逻辑分辨率，默认 `1200x720`。
- 固定截图宽高，默认 `1200x720`。
- 水平和垂直裁剪偏移，用于微调 Poi 游戏画面位置。
- 点击延迟和鼠标移动耗时。
- OpenCV 模板匹配阈值。
- 出击阵型。
- 最大出击轮次。
- 大破停止。
- 中破撤退。
- 停止快捷键。
- 实时预览开关、刷新间隔和鼠标位置高亮。

保存配置后，新截图、新单步执行和新任务会立即使用最新配置。

## 停止快捷键

默认停止快捷键是：

```text
Ctrl+Shift+S
```

可以在 GUI 左侧修改后点击“保存配置”。快捷键通过 `pynput` 注册为全局热键，通常在 Poi 窗口获得焦点时也能停止脚本。

如果热键注册失败，日志会显示原因；仍然可以点击 GUI 上的“停止”按钮。

## 截图区域

默认配置：

```yaml
game:
  logical_width: 1200
  logical_height: 720
  capture_width: 1200
  capture_height: 720
  offset_x: 0
  offset_y: 0
  crop_mode: "left_center_fixed"
```

程序会查找选中的 Poi 浏览器窗口，在窗口客户区内从左侧开始、垂直居中裁出固定 `1200x720` 游戏画面。这样可以避开 Poi 顶部菜单栏、右侧数据面板和底部非游戏区域。

裁剪公式：

```text
x = client.left + offset_x
y = client.top + (client.height - capture_height) / 2 + offset_y
```

如果 Poi 的游戏画面有几像素偏差，可以在 GUI 里调整 `offset_x` 或 `offset_y` 后保存配置。目标窗口客户区小于固定截图区域时，程序会报错并停止截图，避免错误缩放造成点击错位。

## 实时预览和鼠标坐标

默认开启实时预览，刷新间隔为 `200ms`。鼠标移动到游戏画面内时，预览图会显示红色十字、圆环和当前逻辑坐标，例如 `520, 310`。

这个坐标就是 `config/tasks/sortie.yaml` 中点击点应填写的 `1200x720` 逻辑坐标。

## 出击规则

出击规则在 `config/tasks/sortie.yaml`。

状态结构示例：

```yaml
home:
  detect:
    type: "always"
  actions:
    - type: "click"
      point: "sortie_entry"
    - type: "wait"
      ms: 800
  next: "map_select"
  timeout_ms: 5000
  on_fail: "stop"
```

支持的识别条件：

- `always`：总是匹配，用于先跑通坐标点击流程。
- `template`：使用 OpenCV 模板匹配。
- `manual_flag`：预留给 GUI 或调试开关。

支持的动作：

- `log`：写日志。
- `click`：点击 `points` 中配置的逻辑坐标。
- `wait`：等待指定毫秒。
- `formation`：点击 `formation_points` 中当前配置阵型。
- `damage_check`：执行血量规则预留逻辑。
- `stop`：停止当前任务。

## 坐标规则

所有坐标都按游戏逻辑分辨率 `1200x720` 填写。

示例：

```yaml
points:
  sortie_entry: { x: 1040, y: 520, name: "出击入口示例" }
```

这些坐标目前是占位示例。正式使用前，需要用截图预览和单步执行确认真实坐标。

## 模板目录

模板图放在 `assets/templates/`。

例如：

```text
assets/templates/sortie/damage/heavy.png
assets/templates/sortie/damage/medium.png
```

规则中这样引用：

```yaml
detect:
  type: "template"
  template: "sortie/damage/heavy.png"
  threshold: 0.88
```

模板缺失时程序不会崩溃，会在日志中提示缺失路径。

## 后续扩展

新增任务建议按下面方式组织：

```text
src/poi_auto/tasks/repair/
src/poi_auto/tasks/resupply/
src/poi_auto/tasks/expedition/
config/tasks/repair.yaml
config/tasks/resupply.yaml
config/tasks/expedition.yaml
```

约定：

- `device/` 只处理窗口、截图和点击。
- `vision/` 只处理模板匹配和识别。
- `tasks/` 只处理游戏流程。
- `gui/` 只处理界面展示和按钮交互。
- `core/` 只处理配置、运行上下文和任务线程。

这样可以逐步加入远征、修理、补给、任务领取等功能，而不会把所有逻辑堆进一个脚本。
